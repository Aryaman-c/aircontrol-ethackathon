# AIR CONTROL — AI Municipal Decision Support Operating System

> **India already has air quality data. The missing piece is an AI system that tells governments *why* pollution is happening, *what will happen next*, and *what action they should take*.**

**AIR CONTROL** is an AI-powered Environmental Decision Support System designed for city administrators, municipal commissioners, and environmental task forces across Indian metros (Delhi NCR, Mumbai, Bengaluru, Chennai).

---

## 🚀 How to Run This Project

### 1. How to Get a Free Google Gemini API Key
1. Visit **Google AI Studio**: [aistudio.google.com](https://aistudio.google.com)
2. Sign in with your Google Account.
3. Click **"Create API Key"** and copy your generated key string.

---

### 2. Where to Configure Your API Key

#### **Option A: Environment Variable (Recommended)**
- **Windows CMD**:
  ```cmd
  set GEMINI_API_KEY=your_copied_api_key
  ```
- **Windows PowerShell**:
  ```powershell
  $env:GEMINI_API_KEY="your_copied_api_key"
  ```
- **macOS / Linux**:
  ```bash
  export GEMINI_API_KEY="your_copied_api_key"
  ```

#### **Option B: Directly in Code (Fallback Default)**
Open `api/_llm.py` and replace `"your_api_key"` on **Line 8**:

```python
# file: api/_llm.py (Line 8)
GEMINI_API_KEY = os.environ.get("GEMINI_API_KEY", "your_api_key")
```

---

### 3. Step-by-Step System Execution

#### **Step 1: Install Dependencies**
```bash
# Install Frontend Packages
npm install

# Install Backend Python Packages
python -m pip install fastapi uvicorn httpx pydantic
```

#### **Step 2: Start the Python LLM Backend (Terminal 1)**
```cmd
set GEMINI_API_KEY=your_api_key
python -m uvicorn api.advisory:app --port 8005 --reload
```
*(Backend active at `http://127.0.0.1:8005`)*

#### **Step 3: Start the Frontend Console (Terminal 2)**
Open a second terminal window in the project root:
```bash
npm run dev
```
*(Frontend active at `http://localhost:8080`)*

#### **Step 4: Open in Browser**
👉 Open **`http://localhost:8080`** to access the complete AIR CONTROL Operating System!

## 🔥 Key Project Features

### 1. Geospatial Multi-Modal Pollution Source Attribution
- Synthesizes satellite plume reflectance, industrial stack sensor feeds, traffic density indicators, and meteorological trapping factors.
- Quantifies exact source shares (Vehicular Traffic, Construction Dust, Industrial Emissions, Waste Burning, Meteorological Trapping) per municipal ward.

### 2. Hyperlocal 24h–72h Predictive Simulator & "What-If" Policy Lab
- Interactive municipal policy sliders:
  - **Halt Construction & Dust Control Compliance**
  - **Heavy Diesel Freight Diversion**
  - **Anti-Smog Water Sprinklers & Road Sweeping**
  - **Upwind Agricultural Stubble Burning Enforcement**
- Recalculates real-time 72-hour AQI curves against persistence baselines.

### 3. Targeted Enforcement Command Console & Inspector Dispatch
- Ranks registered facilities by violation probability.
- Ingests satellite thermal anomalies (FIRMS), PM10 sensor spikes, and permit status to generate official inspection squad dispatch orders.

### 4. Ask the Reasoning Layer (Google Gemini LLM Engine)
- Natural language Q&A interface for municipal officers.
- Officers can ask questions like *"Why is AQI rising in Anand Vihar?"*, *"Which source is most responsible?"*, or *"Where should we send inspection teams?"*, obtaining dynamic, evidence-grounded responses generated by Google Gemini (`gemini-flash-latest`).

### 5. Population Vulnerability Mapping & Multilingual Advisory Engine
- Maps sensitive populations (schools, hospitals, outdoor workers).
- Generates 2-sentence public health advisories in native scripts (**Hindi**, **Kannada**, **Tamil**, **Marathi**, **English**).

### 6. Interactive Geospatial Command Map
- Full Leaflet dark command canvas rendering Indian CAAQMS ground monitoring stations (Anand Vihar, RK Puram, Punjabi Bagh, Chembur, Peenya, Manali) with dark/satellite view toggles.
